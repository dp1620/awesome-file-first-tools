# Awesome Markdown & File-First Tools [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

> A curated list of high-quality open-source tools built around Markdown, plain text, and file-first workflows.
>
> Only projects that are **open source**, **actively maintained**, and have **200+ GitHub stars** are included.

Markdown is more than a writing format. In the best tools, it becomes an interface: for notes, docs, publishing, APIs, slides, diagrams, notebooks, and developer workflows. This list focuses on tools where files are first-class, Git works naturally, and users keep ownership of their content.

## Editors

- [StackEdit](https://github.com/benweet/stackedit) - In-browser Markdown editor with offline support and syncing.
- [Mark Text](https://github.com/marktext/marktext) - Open-source Markdown editor with a clean writing experience.
- [Zettlr](https://github.com/Zettlr/Zettlr) - Markdown editor for long-form writing, research, and publishing.
- [ReText](https://github.com/retext-project/retext) - Lightweight desktop Markdown editor with live preview.
- [HedgeDoc](https://github.com/hedgedoc/hedgedoc) - Real-time collaborative Markdown editor for teams.
- [Markdown Here](https://github.com/adam-p/markdown-here) - Write Markdown in the browser and render it anywhere.

## Knowledge & Notes

- [Logseq](https://github.com/logseq/logseq) - Privacy-first knowledge base built on local Markdown and org-mode files.
- [Joplin](https://github.com/laurent22/joplin) - Note-taking app with Markdown support and local-first workflows.
- [Foam](https://github.com/foambubble/foam) - Personal knowledge management system built on VS Code and plain-text notes.
- [Memos](https://github.com/usememos/memos) - Self-hosted note-taking tool for quick capture and lightweight knowledge management.

## Inclusion Criteria

Projects must:

- be **open source**
- have **200+ GitHub stars**
- be **actively maintained**
- use **Markdown, plain text, or source files as a core part of the workflow**
- treat files as **first-class inputs**, not just export formats
- have **real-world utility** for developers, teams, writers, researchers, or technical creators
- fit naturally into **local-first**, **Git-friendly**, or **developer-owned** workflows

Projects should not be added if they are:

- closed source
- abandoned or effectively unmaintained
- thin wrappers, demos, or trivial clones
- GUI-first apps where Markdown or files are only a minor feature
- low-signal repos with unclear real-world utility

## Contributing

Contributions are welcome.

Before submitting a pull request, please make sure that the project:

- meets all inclusion criteria
- is open source
- has at least 200 GitHub stars
- fits the scope of Markdown-first or file-first workflows
- is added to the most relevant category
- is not already listed

Please keep descriptions short, factual, and consistent with the style of the list.
