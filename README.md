# Awesome Markdown Dev Tools [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

> A curated list of high-quality open-source developer tools built around Markdown, plain text, and file-first workflows.
>
> Only projects that are **open source**, **actively maintained**, and have **200+ GitHub stars** are included.

This list focuses on developer tools where files are first-class: API workflows, technical docs, notebooks, slide decks, diagrams, and component-driven content.

## API & Developer Workflows

- [Voiden](https://github.com/VoidenHQ/voiden) - Git-native API IDE where requests, docs, and workflows live in Markdown files.
- [MDX](https://github.com/mdx-js/mdx) - Combine Markdown with JSX for interactive developer content and docs.

## Documentation & Publishing

- [mdBook](https://github.com/rust-lang/mdBook) - Build books and technical documentation directly from Markdown.
- [Quarto](https://github.com/quarto-dev/quarto-cli) - Open-source publishing system for technical docs, books, websites, and reproducible content.

## Presentations

- [Slidev](https://github.com/slidevjs/slidev) - Developer-first slide decks written in Markdown.
- [Marp](https://github.com/marp-team/marp) - Markdown presentation ecosystem for writing and exporting slide decks.

## Diagramming

- [D2](https://github.com/terrastruct/d2) - Diagram scripting language that turns text into diagrams.
- [Mermaid](https://github.com/mermaid-js/mermaid) - Generate diagrams like flowcharts and sequence diagrams from text.

## Notebooks

- [marimo](https://github.com/marimo-team/marimo) - Reactive Python notebooks stored as pure Python files and designed for reproducible workflows.

## Inclusion Criteria

Projects must:

- be **open source**
- have **200+ GitHub stars**
- be **actively maintained**
- be clearly useful for **developer workflows**
- use **Markdown, plain text, or source files as a core part of the workflow**
- treat files as **first-class inputs**, not just export formats

Projects should not be added if they are:

- closed source
- abandoned or barely maintained
- GUI-first tools where files are only a minor feature
- thin wrappers, demos, or trivial clones
- general productivity apps without a strong developer workflow angle

## Contributing

Contributions are welcome.

Before opening a pull request, please make sure the project:

- meets all inclusion criteria
- is developer-tool specific
- is open source
- has at least 200 GitHub stars
- is not already listed

Please keep descriptions short, factual, and consistent with the style of the list.
